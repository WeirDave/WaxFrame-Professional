What gets rewritten?

For each ugly ID found in LS_HIVE.customAIs[], the cleaner derives a clean ID from the endpoint URL (e.g. api.mistral.ai → mistral) and rewrites every cross-reference inside the file:

• LS_HIVE.activeAIIds[]
• LS_HIVE.builder
• LS_HIVE.keys (object keys)
• LS_HIVE.models (object keys)
• LS_HIVE.customAIs[].id + .provider + .name
• LS_HIVE.customAIConfigs (object keys + each entry's .label)
• IDB_SESSION.sessionAIs[]
• IDB_SESSION.ringBuffer[].ai + .provider + .aiId

Collisions get an (N) suffix — same logic as WaxFrame's in-app makeCleanProviderId. The cleaner is idempotent: running it on a clean file reports zero changes. DeepDive capture files are flagged as a different format and skipped.
