WaxFrame User Manual

WaxFrame puts a team of AI assistants to work on your document at the same time. One AI writes and refines — the rest review, challenge, and suggest improvements. Round by round, your document gets sharper. This manual walks you through every screen in the exact order you will use them, then covers advanced features and reference material in the appendices.

What WaxFrame Does
The short version

WaxFrame runs multiple AI assistants on your document at the same time. Some act as reviewers — they read your document and give specific, numbered suggestions. One acts as the Builder — it reads all the suggestions and rewrites the document incorporating the best ideas. Each time you run a round, the document improves. You decide when it is good enough.

You can start from a blank page, paste in existing text, or upload a file. WaxFrame works on any kind of written document — proposals, reports, emails, résumés, blog posts, marketing copy, technical documentation, and more. It works equally well whether you are creating something new or improving something you already have.

The three screens
ScreenWhat you do there
SetupConnect your AI assistants by entering their API keys. Pick which AI will act as the Builder. This only needs to be done once — your settings are saved.
ProjectName your project, write your goal, and tell WaxFrame what to start with — an uploaded file, pasted text, or nothing at all.
WorkRun rounds, review the updated document, resolve any conflicts between AIs, and export when you are done.
What you will need before you start

WaxFrame connects to AI providers using API keys — unique codes that give WaxFrame permission to use each AI on your behalf. You will need at least two API keys from any combination of the supported providers: ChatGPT (OpenAI), Claude (Anthropic), Gemini (Google), Grok (xAI), Perplexity, or DeepSeek. You pay each provider directly for what you use — WaxFrame itself is free for your first 3 rounds.

Not sure which AIs to start with? ChatGPT and Claude are both reliable, widely available, and handle all document types well. DeepSeek is an excellent low-cost option. You only need two to get started.

What is index.html and why does it open in a browser?

WaxFrame is a browser-based application — it runs entirely inside your web browser with no installation required. Double-click index.html to open it. All your settings, API keys, and session data are saved in your browser's local storage — not in the folder itself. This means the files in the folder do not change as you use the app.

⚠️ Clearing your browser's site data or cache will erase your API keys and settings. Use Menu → Backup Session regularly to save a copy you can restore.

Bookmark it for easy access

Once you have index.html open, bookmark it in your browser (Ctrl+D on Windows, Cmd+D on Mac). On Windows you can also right-click the file and choose Send to → Desktop to create a shortcut. On Mac, drag the file to your Dock.

Step 1 — Set Up Your Worker Bees
What you are looking at

The Setup screen is the first screen after the welcome page. The left side — Your Worker Bees — is where you connect your AI assistants by entering their API keys. Each AI with a saved key participates as a reviewer in every round, reading your document and giving numbered suggestions.

You need at least two AIs with saved API keys before WaxFrame will let you continue. There is no maximum.

What is an API key?

An API key is a unique code that gives WaxFrame permission to use an AI on your behalf. Think of it as a password between WaxFrame and the AI provider. Each provider issues its own key — you get one from OpenAI for ChatGPT, a separate one from Anthropic for Claude, and so on. Your keys are stored only in your browser and are never sent anywhere except directly to the provider when a round is running.

Need help getting API keys? Click API Key Guide at the top of the Worker Bees panel for step-by-step instructions and direct links to each provider's key console.

Entering your API keys
  1. 1Find the AI you want to use in the Worker Bees list — for example, ChatGPT or Gemini.
  2. 2Click the API Key field next to that AI.
  3. 3Paste your key and press Enter. The field confirms the key is saved.
  4. 4Repeat for at least one more AI. You need a minimum of two.
Testing your keys

After entering a key, click the Test button next to that AI. WaxFrame sends a minimal test request and opens a result panel showing the exact endpoint used, what was sent, the HTTP status code and response time, and the raw response from the server. A green checkmark means the key is working. A red X means something is wrong — the panel will show the error and a plain-English explanation of what caused it.

To test all your saved keys at once, click Test All Keys at the top of the panel. Each AI is tested in sequence and a results panel appears. Click ▶ Details on any row to see the full send and receive for that AI.

Always test your keys before your first round. A bad key will cause that AI to fail mid-round and its suggestions will not be included.

Managing your Worker Bees

Each AI row has several controls. Hide removes an AI from your active hive without deleting its key — useful if you want to temporarily exclude it. Remove deletes the AI and its key entirely. Open API Websites opens the billing or key console for every AI in your hive simultaneously. Reset to Defaults restores any AIs you previously hidden or removed.

Want to connect an AI that is not in the default list — such as a local model, a third-party provider, or your company's internal AI platform? See Appendix A — Adding a Custom AI.

Step 2 — Choose Your Builder
What the Builder does

The Builder is the AI that actually rewrites your document each round. While your reviewers give numbered suggestions, the Builder reads all of those suggestions plus the full document, then produces the updated version. Choose an AI you trust with large documents — the Builder carries more responsibility than any individual reviewer.

Because the Builder handles the entire document every round, it uses significantly more tokens than reviewers. The Builder must have a paid API subscription with sufficient capacity. Do not use a free-tier AI as your Builder — it will run out of tokens quickly and cause rounds to fail.

How to choose your Builder
  1. 1Look at the Builder section on the right side of the Setup screen.
  2. 2Click the AI you want to use as your Builder. Only AIs with a saved key appear here.
  3. 3The selected AI is highlighted with a crown icon. You can change your Builder at any time, including from the Work screen during a session.

Claude, ChatGPT, Gemini, and DeepSeek all handle large documents reliably. DeepSeek is particularly cost-effective. If a round fails with a structure error, switch your Builder to ChatGPT or Gemini and retry. See What Are Tokens? for cost guidance.

Ready to continue

Once you have at least two API keys saved and a Builder selected, the Continue to Project Setup → button at the bottom of the screen activates. Click it to move to the Project screen. Your keys and Builder selection are saved automatically.

Step 3 — Define Your Project
What you are looking at

The Project screen has two sides. The left side — Your Project — is where you name your project and write your goal. The right side — Starting Document — is covered in Step 4. Fill in the left side first, working top to bottom.

Project Name and Version Number

Enter a name for your project in the Project Name field — for example, Q3 Executive Summary or Cover Letter — Senior Engineer. Enter a version in the Version field — for example, v1. These appear in the export filename and in the round history log. They do not affect how the AIs work.

Length Constraint (optional)

If your document must fit within a specific limit — for example a 500-word blog post, a 200-character social media post, or a one-page résumé — enter the limit here and select the unit: characters, words, or pages. The constraint is included in every prompt sent to your AIs and they will actively work to stay within it. Leave it blank for documents with no length requirement.

Export Filename

This controls what your downloaded document file will be named. It uses tokens that are replaced automatically at export time. The default — {name}_{version} — produces a filename like Q3_Executive_Summary_v1.txt. You can customise it or leave it as the default. Click the ⓘ icon next to the field for a full explanation of available tokens.

The Project Goal — the single most important thing you will write

Every AI reads your goal before touching your document. It tells them what the document is, who it is for, what it should and should not include, and what outcome you need. The goal works the same way regardless of whether you are starting from scratch, uploading a file, or pasting text. The difference is only what the AIs have to work with — not how the goal is used.

A vague goal produces a vague document. A specific, detailed goal produces something you can actually use.

Cover in your goalGuidance
What the document isState the format clearly — a proposal, a comparison, a cover letter, an executive summary. Be specific about type and purpose.
Who will read itYour IT team? A client? A hiring manager? The audience shapes the tone and depth every AI will use.
Scope — what to include and what to excludeIf the document covers one topic only, say so. If certain content should not appear, state that explicitly. AIs expand into adjacent topics unless you draw clear boundaries.
What you want the reader to doMake a decision? Take an action? Understand a concept? Give the AIs a finish line.
Tone and styleFormal or conversational? Technical or plain-English? First person or third? The more specific you are, the more consistent the output will be across rounds.
Writing a goal when you already have a document

This is one of the most common ways to use WaxFrame — you have a document that exists and you want the hive to make it better. Your goal still matters, but your focus shifts from what to create to what to improve.

A strong refinement goal answers: What is this document and what is it trying to achieve? What is working well and should be left alone? What specifically needs to improve — clarity, structure, tone, argument, length? What must not change under any circumstances?

Too vague "Make this document better."

Much better "This is a proposal for a wireless network infrastructure upgrade at a 950,000 sq ft warehouse facility. The audience is the IT Director and VP of Facilities. The technical sections and the vendor comparison table are solid and should not be changed. Focus improvements on the executive summary — it is currently too long and too technical for a VP audience — and on the cost justification section, which needs stronger ROI language. Tone should be confident and direct. Do not add any new sections."

Writing a goal when starting from scratch

When you have no existing document, your goal is the only context the AIs have. Be as detailed as possible — include the document type, audience, purpose, scope, tone, approximate length if known, and any specific sections or structure you want.

Too vague "Write a report about wireless networking."

Much better "Create a technical comparison of enterprise wireless networking platforms — HPE Aruba, Cisco Meraki, Juniper Mist, and Extreme Networks. Audience is our IT infrastructure team evaluating a campus-wide refresh. Scope is strictly wireless: access points, controllers, cloud management, and licensing. Do not include switching, routing, or security products. Tone is technical and neutral. Include a summary comparison table at the top followed by a section for each vendor. The output should help the team produce a shortlist recommendation."

Not sure how to write your goal? The Document Playbooks has ready-made goals for common document types — résumés, proposals, blog posts, RFPs, and more. Open it from the Menu.

How your goal is used — Draft round vs Refine rounds

WaxFrame uses your goal differently depending on which round you are on.

Draft round Your full goal is sent to every AI regardless of length. This is when the goal matters most — the AIs are producing the first version of your document and need all the context you can give them.
Refine rounds A trimmed version of your goal is sent, labelled Project Context. The trim targets approximately 300 characters and always cuts at a sentence boundary so nothing is left mid-thought. Your document already exists — the tokens are better spent on the document content and reviewer suggestions.

Put your single most important instruction in the first sentence of your goal. That sentence will always be included in Refine rounds. The character counter next to the goal field turns amber when you exceed 300 characters.

The Refine Preview — see exactly what your AIs will receive

The ▸ Refine Preview button next to the character counter opens a preview of the exact text that will be sent to your AIs in Refine rounds, updated live as you type. If your goal is under 300 characters, the preview shows your full goal and confirms it will be sent in full. If your goal is over 300 characters, the preview shows the trimmed version cut to the nearest sentence boundary. Use it to verify that your most critical instruction is included within the limit.

On screens with limited height the Refine Preview panel collapses — the ▸ Refine Preview button is always available to check what your AIs will receive.

Step 4 — Provide Your Starting Document
What you are looking at

The right side of the Project screen — Starting Document — is where you tell WaxFrame what to work with. The three buttons at the top — Upload File, Paste Text, and Start from Scratch — are mode selectors, not action buttons. Click one to choose your method, then follow the instruction that appears below the buttons.

Upload File — I have a document saved on my computer

Use this option when you have an existing document saved as a file. Click the Upload File button to select that mode. A drop zone appears below — click anywhere inside it to open your file browser, or drag and drop your file directly onto it.

Supported formats: Word (.docx), PDF (.pdf), PowerPoint (.pptx), plain text (.txt), and Markdown (.md). WaxFrame extracts the text from your file and loads it into the working document automatically. A confirmation message appears once the file is processed successfully.

PDF tip: PDF extraction works best with standard digitally-created files — documents exported from Word or Google Docs. Scanned documents or PDFs built from images may produce garbled or incomplete text. If that happens, copy the text from your PDF and use the Paste Text option instead.

⚠️ If a round fails with a structure error after importing a very large file, switch your Builder to ChatGPT or Gemini using Change Builder in the Work screen and retry. Some AIs handle very large documents more reliably than others.

Paste Text — I have text I want to copy in

Use this option when you have existing text you want to paste — from a website, another application, an email, or anywhere else. Click the Paste Text button to select that mode. A text editor appears below it. Click inside the editor and paste your text using Ctrl+V on Windows or Cmd+V on Mac. You can also type directly into the editor.

The editor uses the same fixed-width 80-character format as the working document. This is intentional and ensures consistent line wrapping throughout your session.

Start from Scratch — I do not have a document yet

Use this option when you have nothing written yet and want the AIs to produce the first draft. Click Start from Scratch to select that mode. Nothing else is needed on this side of the screen. The hive will generate the first draft based entirely on your project goal — so your goal needs to be as detailed as possible before you launch.

Launch WaxFrame

The requirements bar at the bottom of the screen shows what is still needed before you can launch — a project name, a version number, a project goal, and a document method selected. All four must show a green checkmark. Once they do, the Launch WaxFrame → button activates. Click it to move to the Work screen and begin your session.

Step 5 — Launch and Run Your First Round
What you are looking at

The Work screen has three columns. The left column — The Hive — shows your AI reviewers and their status during a round. The center column — Working Document — shows your document as it evolves. The right column shows individual reviewer responses and any conflicts that need your decision.

Click "Smoke the Hive" and let it run

Click the Smoke the Hive button at the bottom of the screen to start a round. All your reviewers run simultaneously — having more AIs does not mean it takes longer. You will see each AI's status update in the Hive panel as it works. Once all reviewers complete, the Builder runs automatically and produces the updated document.

Do not close the browser or navigate away while a round is running. If a round is interrupted, the document will not be updated and the round will not be saved to history.

What happens in the first round

Your first round is always a Draft round. Your full project goal is sent to every AI. If you chose Start from Scratch, each AI produces a first draft. If you provided a document via upload or paste, each AI reviews it and produces numbered suggestions. The Builder then reads all drafts or suggestions, combines the best elements, and produces the first working document.

Every subsequent round is a Refine round. The AIs read the current document and give targeted improvement suggestions. The Builder incorporates the strongest suggestions and produces an updated version. Your goal is sent as a trimmed Project Context in all Refine rounds.

Step 6 — Review Results and Resolve Conflicts
Reading the updated document

After a round completes, the center column shows the updated document. Read it carefully. Each completed version is saved automatically to your round history — click History in the top bar to view or restore any previous version.

What is a conflict?

A conflict appears when your AIs disagreed on something significant enough that the Builder flagged it for your decision rather than choosing arbitrarily. Conflicts appear in the right column after a round completes. Examples: one AI suggested removing a section while another suggested expanding it; AIs disagreed on tone or phrasing for a key passage; reviewers proposed incompatible structural changes.

How to resolve a conflict

Each conflict shows the original text and two or more options — the different directions your AIs proposed. Read each option and click the one that best matches your intent. You can also select Custom and type your own resolution, or Keep Original to leave that section unchanged. Once you have resolved all conflicts, click Apply Decisions to update the document.

Resolved conflicts are locked

Once you apply a decision, it is locked. WaxFrame informs all your AIs about resolved decisions in future rounds and instructs them not to re-raise the same issue. If an AI keeps re-raising a resolved conflict despite this, use the Notes drawer to give the Builder a direct instruction to enforce it.

Using conflicts to steer your document

Conflicts are not a sign something went wrong — they are a signal that your AIs found a genuine decision point. Use them deliberately. If you are unsure which direction to take, read each option carefully. The AIs are often surfacing a real question about structure, argument, or tone that you will need to answer to get the document where you want it.

Step 7 — Run More Rounds
Each round improves on the last

The process is the same every round: click Smoke the Hive, wait for it to finish, read the updated document, resolve any conflicts, and decide whether to run another round. Most documents reach a strong state in 3 to 5 rounds. Some need more, some need fewer.

Using Notes to give the Builder specific instructions

The Notes drawer — accessible from the top right of the Work screen — lets you give the Builder direct instructions for the next round only. Use it when you want something specific: "Tighten the third paragraph considerably," or "The conclusion is weak — rewrite it to be more decisive," or "Do not add any more bullet points." Notes are cleared after each round so they only apply once.

Editing the document directly between rounds

The working document is editable at any time between rounds. If you can see exactly what needs to change, edit it directly. The AIs will refine what is there in the next round. If you remove something and mention the removal in your Notes, the AIs are much less likely to reintroduce it.

Adjusting which AIs run each round

Click Edit Hive in the Hive panel to toggle individual AIs on or off for the current session. Turning an AI off does not delete its key — it just skips that AI for the next round. Use this to exclude an AI that is giving poor suggestions, or to speed up a round by using fewer reviewers.

Knowing when you are done

Watch for these signals: the AIs start agreeing more than disagreeing; conflicts become minor word choices rather than structural issues; you find yourself accepting most suggestions without hesitation; the document reads the way you want it to. When any of these apply, you are probably done. Running more rounds past this point produces diminishing returns.

Step 8 — Export and Finish
Exporting your document

Click 🏁 Finish in the top right of the Work screen to open the Finish panel. From here you have three options.

Export Document Downloads your working document as a .txt file using the export filename you set on the Project screen. Use this when you just need the final text.
Export Full Transcript Downloads every round, every AI response, all conflicts and decisions, and the final document in a single file. Use this for a full audit trail or to review what each AI contributed.
Save Session Snapshot Saves your entire session — API keys, project details, document, and round history — as a .json file. Use this to pause and resume later without losing your work. Load it back via Menu → Import Backup.
Starting a new project

Click Start New Project in the Finish panel to clear the current session and begin again. Your API keys and Builder selection are preserved. Your working document and round history are cleared permanently.

⚠️ Starting a new project cannot be undone. Export your document and save a Session Snapshot before clicking Start New Project if there is any chance you will want to return to this session.

Appendix A — Adding a Custom AI
When would you add a custom AI?

WaxFrame comes pre-configured for six public AI providers. If you want to connect a provider not in the default list — such as Mistral, Together AI, Cohere, or Ollama running locally — you can add it using the Add Custom AI form. Any AI that exposes an OpenAI-compatible API, the Anthropic API, or the Google API can be connected.

To open the form, go to the Setup screen and click Add Custom AI at the top of the Worker Bees panel.

The Add Custom AI form — field by field
FieldWhat to enter
Quick AddOptional. Presets for Mistral, Together AI, Cohere, Ollama, and LM Studio. Selecting one auto-fills the endpoint URL and API format. Fill in fields manually if your provider is not listed.
Display NameWhat WaxFrame calls this AI in the UI. Auto-filled by Quick Add. You can change it to anything you like.
API Endpoint URLThe full address WaxFrame sends requests to, including the path. For most OpenAI-compatible providers this ends in /v1/chat/completions. Example: https://api.mistral.ai/v1/chat/completions
API FormatOptional if using Quick Add — auto-filled. Otherwise select OpenAI compatible for most providers, Anthropic for Claude endpoints, or Google for Gemini endpoints.
ModelThe exact model ID string the provider expects. Must be exact — even a minor typo causes every call to fail. Use Fetch Models to retrieve the list directly from the provider rather than typing it manually.
API KeyYour key or bearer token for this provider. Paste it in and press Enter to save.
Fetch Models

Click Fetch Models after entering your endpoint URL and key. WaxFrame queries the provider's models endpoint and returns a list of available models. Select the one you want from the dropdown that appears. This is more reliable than typing a model ID manually and eliminates typo errors.

Test Connection — always do this before adding

Click Test Connection to fire a real minimal API call before adding the AI to your hive. WaxFrame shows you the exact endpoint used, the JSON body that was sent, the HTTP status code and response time, and the complete raw response received from the server. If the test fails, a plain-English explanation is shown:

  • 401 / 403 — bad or missing API key
  • 404 — wrong endpoint URL
  • 405 — method not allowed (endpoint may not support chat completions)
  • 429 — rate limited (wait a moment and try again)
  • Network Error — the endpoint is unreachable or a CORS policy is blocking the request

⚠️ The Add to Hive button only appears after a passing test. If you change any field after a passing test, the result resets and you must test again before adding.

Appendix B — Importing from a Model Server
What is a model server?

A model server is software that hosts AI models locally on your machine or network — for example, Open WebUI, Ollama, or LM Studio. Instead of sending requests to a cloud provider, requests go to a server running on your own hardware. The Import from Model Server feature lets you fetch the full list of models available on your server and add multiple AIs to your hive at once.

How to import from a model server
  1. 1Go to the Setup screen and click Import from Model Server.
  2. 2Use the Quick Add dropdown to select your server type — Open WebUI, Ollama, or LM Studio. This auto-fills the endpoint fields. Fill them in manually if your server is not listed.
  3. 3Confirm or enter your Chat Endpoint and Models Endpoint URLs.
  4. 4Enter your API key or bearer token if your server requires authentication.
  5. 5Click Fetch Models. WaxFrame queries your server and returns the list of available models.
  6. 6Click Select & Add to Hive →, check the models you want to add, and click Add Selected to Hive.

If Fetch Models returns a CORS error, your server may need to be configured to allow requests from file:// origins, or you may need to run WaxFrame from a local web server. Ask your IT team for assistance with this configuration.

Appendix C — Using WaxFrame at Work
WaxFrame works with internal enterprise AI platforms

Many organisations run their own AI platforms — tools like Open WebUI connected to internally hosted models, or enterprise subscriptions to OpenAI or Anthropic with private endpoints and access controls. WaxFrame can connect to these using the Add Custom AI or Import from Model Server features, provided the platform exposes an API endpoint that accepts chat completion requests.

What to ask your IT team for
What you needWhat to ask for
API endpoint URL"What is the chat completions endpoint URL for our AI platform?"
API key or token"How do I authenticate API requests — is it a bearer token, an API key, or something else?"
Model name"What is the exact model ID string I should use in my API calls?"
API format"Is the API OpenAI-compatible?" (The answer is almost always yes for enterprise platforms.)
Setting up an internal-only hive

If your organisation uses internal models only and you do not want the public AI options visible, hide them individually using the Hide button on each AI row in the Setup screen. Hidden AIs are excluded from all rounds and the setting is remembered across sessions. A banner at the top of the Worker Bees list shows how many are hidden, with a ↺ Restore Hidden button to bring them back.

CORS and running WaxFrame from a local server

WaxFrame runs as a local HTML file in your browser. Some internal AI servers block requests from local file origins as a CORS policy. If you see a Network Error or CORS error when testing a connection to an internal endpoint, ask your IT team to allow requests from file:// origins on the AI server, or to serve WaxFrame from an internal web server so requests originate from a recognised HTTP address.

When Things Go Wrong
The document is going in the wrong direction
SituationWhat to do
The tone is wrongOpen the Notes drawer before the next round and write: "The tone throughout must be [formal / conversational / technical]. Adjust the entire document." Be specific about what you want.
The AIs keep adding things you do not wantEdit your goal to explicitly exclude the unwanted content. Reinforce the exclusion in a Note. If one specific AI keeps adding it, use Edit Hive to remove that AI for a round.
The document has drifted badlyOpen History and restore an earlier version. Add a Note explaining what direction the next round should take. If the goal was too vague, rewrite it before running another round.
Starting fresh is fasterUse Clear Project on the Project screen. This clears the document and round history but preserves your API keys. Rewrite your goal with tighter scope and start again.
A round failed with a structure error

This means the Builder returned output in a format WaxFrame could not parse — usually because the document was too large or complex for that AI to follow the required output format precisely. Click Change Builder in the Work screen top bar, switch to ChatGPT or Gemini, and retry the round. Your document was not changed by the failed round.

An AI keeps re-raising a conflict you already resolved

Use the Notes drawer before the next round and write: "[AI name] keeps suggesting [X]. This decision is final — enforce the resolved text and do not allow changes to [Y]." If the problem persists, use Edit Hive to remove that AI from the session.

An API key is not working

Click the Test button next to the AI on the Setup screen. The result panel shows the exact error. Common causes: the key was entered incorrectly (check for extra spaces or missing characters), the key has been revoked or expired at the provider, the account has run out of credit, or the account is on a free tier that does not include API access.

You cannot start a round

WaxFrame requires at least two AIs with saved keys and a Builder selected before the Smoke the Hive button activates. Go back to the Setup screen and verify that at least two API key fields show a saved key and that a Builder is selected on the right side of the screen.

A provider link or signup page has moved

Provider websites change their navigation regularly. If a link in WaxFrame leads to a dead page, open the API Key Guide from the Menu — it is updated with each WaxFrame release and contains current direct links to each provider's key console.

API Costs & Billing
You pay the AI providers directly — not WaxFrame

WaxFrame itself is free to use for the first 3 rounds; after that a one-time license fee applies. The API calls WaxFrame makes are billed directly to your account at each provider. Costs are measured in tokens — small units of text. A round involving a medium-length document and four AIs typically costs between $0.05 and $0.30 depending on the models used.

DeepSeek is significantly cheaper than the major US providers and produces high-quality output. It is an excellent choice for the Builder role when cost is a consideration. See What Are Tokens? for a full cost breakdown.

Monitor your billing regularly

Provider pricing changes. Check your usage dashboard at each provider periodically. Click Open API Websites on the Setup screen to open all your provider billing consoles at once. Most browsers will block multiple tabs opening simultaneously — allow pop-ups for WaxFrame when prompted.

Perplexity — enable auto-billing before use

Perplexity requires you to explicitly enable auto-billing in your account settings before API calls will succeed. If Perplexity tests fail with a billing error, log into your Perplexity account, enable auto-billing, then retry.
