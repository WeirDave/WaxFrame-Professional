The lesson I want to land:

answer. The answer comes from walking the floor with a
laptop and a real survey adapter and finding out where the
model lied to you.

Question to end on: what's the worst predictive-vs-actual
delta you've measured?

──── SETUP 4 — REFERENCE MATERIAL (paste into Setup 4) ─────
Thesis: any single LLM, no matter how good, has blind spots.
Run the same document through 4 or 5 different models in
sequence and the output is meaningfully better than any one
model can produce on its own.

Concrete example to reference:
- Took an Aruba ArubaOS 8.x configuration document, ~900
  words, describing an ArubaOS firmware-upgrade procedure
  for a multi-controller deployment
- Ran it through Claude alone: got a tightened version,
  maybe 15% better. Still missed two ordering issues that
  would have caused a controller failover during the upgrade
- Ran the same document through Claude → ChatGPT → Gemini →
  back to Claude: got a version where both ordering issues
  were caught, the prerequisites section was reorganized,
  and one assumption about RADIUS server reachability was
  flagged as needing verification before the change window

What the reader should try today: pick a runbook or
configuration document they care about, hand it to 3
different models in sequence with the same prompt, compare
the result against running it through any one model once.

Call to action: link to a tool that automates this — multiple
models, single document, convergence loop. (This is WaxFrame
but the post should not name it overtly — let the link
speak for itself.)
