Open Features

- GitHub Actions release-ceremony consistency check — workflow under .github/workflows/ that runs `node --check` on all JS, verifies the four version stamps match across files (meta waxframe-build / APP_VERSION / BUILD / app.js?v=), and validates no dead local asset links, on every push. Automates the manual release verification.

- Pause/resume at reviewer seam — finer-grained pause point INSIDE a round, between reviewers finishing and the builder firing. Current pause/halt mechanics all operate at round boundaries (Auto chain pauses at end of round). The reviewer-seam pause would let the user inspect reviewer outputs, edit notes, and decide whether to run the Builder for this round — without having to wait for the Builder to finish + then halt.

- File System Access API exploration — investigate replacing download-only backups with true file persistence via window.showSaveFilePicker / showOpenFilePicker / FileSystemHandle. Would let a user "Save backup" once and then "Save" repeatedly to the same file without the download-folder pile-up. Browser-support check (Chrome/Edge yes, Safari/Firefox no) gates whether this is additive or a hard cutover.

- Hive Profiles — saved named hive configurations (research / draft / polish / Work Laptop / Home Desktop / etc.). What remains: profile save/load layer on top of the existing selection state — a small additive feature once the data shape is decided.
