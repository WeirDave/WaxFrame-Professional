WaxFrame Storage Schema v1
2026-06-12

ABOUT THIS DOCUMENT
────────────────────────────────────────────────────────────
First-pass inventory of every localStorage key written or
read by WaxFrame Professional. Produced for backlog item #7
(localStorage migration audit). Built from an AST scan of
every js/*.js file plus every inline <script> block in the
HTML helper pages — 224 setItem/getItem/removeItem
operations across 4 JS files and 2 HTML files, normalized
to 42 unique keys (29 static + 13 dynamic-name patterns).

The static enumeration found ZERO orphan keys. Two patterns
look orphan to a static reference scan but are intentional:

  • waxframe_v2_hive_recovery — write-only safety net.
    When loadSettings() can't parse LS_HIVE, the raw blob
    gets stashed here for the user to recover via DevTools.
    Read by humans, not by code. See storage.js:874.

  • waxframe_v2_settings — read-once, removed-once. The
    pre-v3.45 single-blob settings key, kept around so a
    long-cold browser profile from before that release
    migrates cleanly on first load. See storage.js:839-855.

What this document does NOT yet do (left as remaining work
on backlog #7):
  • Manual test plan for forwarding a v3.21-era profile
    through every release boundary. Needs runtime testing
    in a controlled browser; static enumeration is silent
    on whether each read path actually tolerates every
    historical shape it might encounter.

Audit script: .claude/scratch/ls-audit.js (gitignored).
Run with `node .claude/scratch/ls-audit.js <project-root>`
after installing acorn + acorn-walk into .claude/scratch/.

CANONICAL KEYS — declared as LS_* on `window` in storage.js
════════════════════════════════════════════════════════════

LS_HIVE  =  'waxframe_v2_hive'
   ─────────────────────────────────────────────────────────
   PURPOSE: AI list + API keys + builder selection +
            knownDefaultIds. Persists across projects.
   SHAPE:   JSON.stringify({
              activeAIIds: string[],
              knownDefaultIds: string[],
              builder: string,
              keys: { [aiId: string]: string },     // API keys
              customAIs: Array<{ id, name, url, ...}>
            })
   WRITTEN: storage.js:776 (saveHive), storage.js:854
            (migration write from LS_SETTINGS), other
            migration paths in saveHive's caller chain.
   READ:    storage.js:835, storage.js:859, app.js:2267
            (and 6 other call sites; all gated on parse-OK).
   MIGRATION: storage.js:861-886 — JSON.parse wrapped in
              narrow try/catch. On parse failure: raw blob
              stashed to waxframe_v2_hive_recovery, toast
              raised, loadSettings bails BEFORE aiList is
              touched. See v3.63.276 comment block.

LS_PROJECT  =  'waxframe_v2_project'
   ─────────────────────────────────────────────────────────
   PURPOSE: Project name/version/goal + docTab + per-project
            UI state (referenceDocs, exportMask, hivePreset).
            Per project.
   SHAPE:   JSON.stringify({
              name: string,
              version: string,
              goal: string,
              docTab: string,                       // active doc-tab id
              referenceDocs: Array<{...}>,
              exportMask: string,
              hivePreset: string,
              ...
            })
   WRITTEN: storage.js:800 (saveProject), and one
            migration-rewrite path.
   READ:    11 sites across app.js + 1 in wf-debug.js.
            All use JSON.parse(... || '{}') with the
            empty-object fallback.
   MIGRATION: tolerates an empty/missing key via the
              '|| {}' fallback. No explicit version
              branches — the schema accretes optional
              fields and old shapes parse as a subset.

LS_SESSION  =  'waxframe_v2_session'
   ─────────────────────────────────────────────────────────
   PURPOSE: Round state snapshot for resume-after-reload.
            Lightweight envelope; the heavy session payload
            (history, docText, consoleHTML) lives in
            IndexedDB under IDB_KEY = 'current'.
   SHAPE:   JSON envelope — see storage.js's saveSession
            function. Includes round counter, current
            phase, last-Builder timestamp.
   WRITTEN: storage.js:324 (saveSession) + the IDB
            mirror chain in _saveSessionChain.
   READ:    storage.js (loadSession entry), app.js
            resume-detection path.
   MIGRATION: storage.js handles v6/v5/v4 envelope
              versions on read — see the IDB
              onupgradeneeded ladder at storage.js:72
              and the loadSession version-branching code.

LS_SETTINGS  =  'waxframe_v2_settings'
   ─────────────────────────────────────────────────────────
   PURPOSE: Pre-v3.45 single-blob settings key. RETIRED.
            Kept readable so long-cold browsers from
            before v3.45 migrate cleanly on first load.
   SHAPE:   (legacy) JSON.stringify({
              activeAIIds, knownDefaultIds, builder,
              keys, customAIs
            })
   WRITTEN: never (RETIRED on write).
   READ:    storage.js:839 only — inside loadSettings'
            migration branch, gated on `!hiveRaw`.
   MIGRATION: storage.js:840-855 reads legacy, rebuilds
              the LS_HIVE shape from it, writes LS_HIVE,
              REMOVES legacy. One-shot.

LS_LICENSE  =  'waxframe_v2_license'
   ─────────────────────────────────────────────────────────
   PURPOSE: License key + activation metadata. Persistent.
   SHAPE:   JSON.stringify({ key: string, ...activation }).
   WRITTEN: 6 sites across license-helper.js + app.js.
   READ:    16 sites — license-check is a hot path on
            most user-facing actions.
   MIGRATION: license-helper.js handles its own version
              compat. No legacy keys observed.

LS_PROMPTS  =  'waxframe_v2_prompts'
   ─────────────────────────────────────────────────────────
   PURPOSE: User overrides for the prompt blocks shipped
            in app.js (draft_scratch, refine, etc.).
            Edited via the Prompt Editor page.
   SHAPE:   JSON.stringify({ [promptKey: string]: string })
   WRITTEN: prompt-editor.html (inline <script>) — that's
            the only writer.
   READ:    app.js:7724 (recommend-model prompt loader,
            for backwards-compat with pre-v3.32.10 saves)
            and app.js:15771 (getPrompt — the canonical
            prompt loader).
   MIGRATION: getPrompt does an empty-object fallback,
            so a missing key returns the in-app default.

SUPPRESS PROMPTS (dynamic keys; managed in bulk)
════════════════════════════════════════════════════════════

Pattern: waxframe_suppress_${promptKey}
   ─────────────────────────────────────────────────────────
   PURPOSE: User has clicked "Don't show this again" on a
            confirmation prompt. Boolean presence (existence
            of the key = suppressed).
   WRITTEN: Per-prompt callers via wfConfirm + per-prompt
            suppressKey.
   READ:    Same wfConfirm path.
   BULK:    The "↺ Restore dismissed prompts" action in
            Settings clears every waxframe_suppress_* key
            in one pass via Object.keys+removeItem (see
            app.js:1085-1090). That bulk-remove uses a
            dynamic key, which is why the audit reports
            it under [DYNAMIC] rather than per literal
            prompt-key — the literal keys never appear
            statically in the code.
