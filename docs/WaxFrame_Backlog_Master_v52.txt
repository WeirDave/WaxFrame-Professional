Observations

- No real 7-day MODELS_CACHE_TTL existed before v3.62.1. Model caches lived forever, refreshed only by explicit recheck. Auto-Update is the first intentional refresh mechanism.

- wfConfirm suppressKey is zero-per-modal-boilerplate. Adding suppressKey: 'waxframe_suppress_xxx' to any informational confirm's opts makes it dismiss-able and reset-eligible. NEVER set suppressKey on a destructive confirm — those are safety brakes.

- Helper menu = strict subset of main menu. Differences should be deletions (things the helper context can't do), never divergences (different items in the same slot). Principle locked in v3.63.1.

- Dismiss-permanently semantic split — two patterns now in the codebase:
  Pattern A (v3.62.2) — informational warnings: checkbox + confirm = persist; checkbox + cancel = does NOT persist (cancel means "still show me next time")
  Pattern B (v3.63.0) — feature opt-outs: checkbox = persist regardless of which button (the checkbox IS the opt-out intent, decoupled from the immediate action)
  Document the choice explicitly whenever adding a new dismiss-permanently surface.

- Hash deep-link infrastructure exists in DOMContentLoaded as of v3.63.1. Currently only #settings is recognized. Future deep-links extend the same dispatcher — pattern is: read hash, dispatch via setTimeout to let active screen paint, clear hash via history.replaceState so reload doesn't re-trigger.

- v3.62.0 originally scoped Auto-Update Models + Checkpoint Backup Nudge alongside the Length Guard / Slow Alerts mirrors. Auto-Update needed a real scheduler; Checkpoint needed per-project suppression. Both split out cleanly (v3.62.1 and v3.63.0). When a release feels like it's growing extra concerns, ship slimmer and split.

- "Create Something" as a section label read like a creative-writing tool, not navigation. Renamed to "WaxFrame App" in v3.63.1. General lesson: nav labels that hint at a verb instead of a destination cause confusion.

- The Sensitive Backup confirm now chains AFTER the Checkpoint Backup Nudge when both are active. Two distinct concerns: (1) do you want a backup now? (2) the backup is sensitive, store it safely. Don't collapse them.
