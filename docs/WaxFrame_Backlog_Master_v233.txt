Observations

- 2026-06-05 — The viewport-vs-screen-resolution confusion is the #1 source of "but my screen is 1920×1080 so why am I blocked" support friction. Surfacing the actual CSS-pixel viewport in the overlay disambiguates the diagnosis at zero cost — no external dependency, no link rot, no second-page-load. Should have been there from day one of the min-screen-overlay; flagged here as a lesson for any future "what is your environment" gating.

- 2026-06-05 — Inline IIFE pattern (script block in the page body, not deferred to app.js) is the correct architectural choice for any feature that needs to work BEFORE the full app loads. The min-screen-overlay overlay is the canonical example: users hitting it can't load the app, so the diagnostic UI has to be self-contained. Same pattern would apply to any "browser not supported", "JS disabled", "cookies blocked" gating in the future.
