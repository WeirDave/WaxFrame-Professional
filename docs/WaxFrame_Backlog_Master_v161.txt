- Model recommender filter-awareness warning — when the recommender's pick has been flagged in the current session's ringBuffer with CONTENT_FILTERED, surface a ⚠️ next to that model in the Change Builder dropdown and in the recommend toast: e.g. "This model has triggered content filters this session — Flash variants are less aggressive". - Builder Failover — primary Builder hits CONTENT_FILTERED / EMPTY_RESPONSE → auto-failover to a configured Secondary Builder for that one round only, then snap back to primary next round. Mirrors how the hive already handles a single reviewer failing. Implementation: "Secondary Builder" pick on the Builder screen, interceptor in runBuilderOnly() that swaps to secondary on CONTENT_FILTERED, completes the round, resets. Lets users stay on cheap providers (Gemini Flash, DeepSeek) as primary with a reliable but pricier secondary (Claude/ChatGPT) ONLY firing on blocked rounds.

- Builder-capability filter at recommendation/selection time — flag models that can't fulfill the Builder role. Filter should run at: (1) Change Builder modal — greyed/warning-badged option with tooltip "May be unable to follow Builder formatting — try Reviewer role instead"; (2) Recommend Models pipeline — skip Builder-incapable models from Builder recommendations entirely. Mechanism shape: a `builderCapable: boolean` flag on the model metadata, populated from a known-bad list initially, augmented over time by capture from ring-buffer evidence (finishReason === "length" on Builder calls + missing %%CONFLICTS_START%% pattern). Same general shape as the existing "Recommend-time filter for instruction-incapable models (e.g. antigravity-preview)" parking-lot item — that's the same pattern at a different layer.
