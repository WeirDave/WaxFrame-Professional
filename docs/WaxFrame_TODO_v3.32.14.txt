ON THE HORIZON — FUTURE ROADMAP
============================================================

[ ] Hive Profiles / saved AI configurations
    A profile is a saved snapshot of activeAIIds plus
    optionally builder. One dropdown at the top of the
    Worker Bee screen:
       "Profile: [Work Laptop ▾] · Save current as new… · Manage…"

    Server-aware behavior:
    - If a profile references AIs from a server that isn't
      reachable (work VPN dropped, home firewall, air-gap),
      surface a warning instead of silently failing on
      round-start
    - "This profile uses 3 bees from Alfredo
       (oui.ai.usgw1.aws.anduril.dev) — currently
       unreachable. Switch profile?"
    - Could auto-detect on profile load via a quick Models
      endpoint ping

    Export/import:
    - Profiles export as JSON — share between laptop and
      desktop
    - API keys travel separately (or get redacted on export
      and re-prompted on import) — security matters here
    - Could ship as part of existing Backup format v3, or
      as standalone .waxprofile files

[ ] Sources / Servers view (companion to Hive Profiles)
    Soft split, not hard.

    - Worker Bee screen stays the unified view — it's where
      you tick what to use
    - A new "Sources" or "Servers" view (or modal) handles
      server-level concerns: add/edit/remove a server, its
      endpoint pair, its API key, test connectivity, see
      which bees came from it
    - Imported Groups becomes "Sources" — earns its keep
      because it's no longer redundant with the bee list,
      it's the parent of bees from that source

    Export/import maps cleanly: export a Source (endpoint
    + key, optional) plus the Profiles that reference it.
    Import a Source on a new machine and the Profiles light
    up. AIs without a Source are direct-API customs.

[ ] MODEL_LABELS data-driven via IDB daily cache
    With MODEL_FALLBACKS as the offline fallback. Backlog
    item from prior planning.

[ ] Model update detection + opt-in swap prompt
    When a provider releases a new top-tier model, prompt
    the user to swap rather than silently keep the old one.

[ ] User-manual guidance on model diversity for
    model-server deployments
    The Recommend-a-Model convergence trap: minimum-two-
    distinct-models rule. Document somewhere in the user
    manual.

[ ] Documentation: convergence principle callout
    "Starting from scratch converges faster than refining
    heavily misaligned documents." Placement TBD —
    README, manual, or playbooks.

DECISIONS LOGGED — NOT GETTING BUILT

[—] Inline style="display:none" migration in index.html
    Originally flagged as v3.32.15. Confirmed against the
    April 27 prior analysis: 16 elements use the JS-toggle
    pattern where app.js reads el.style.display directly
    (5 read sites on customAIModelSelect alone). Migrating
    to a class-based hide would require auditing and
    rewriting every style.display read across the codebase
    — non-surgical. The inline declarations are kept and
    documented as the project's JS-toggle pattern, not as
    rule violations. Leave alone unless ready to migrate
    every read site.
