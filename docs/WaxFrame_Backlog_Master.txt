2. ARCHITECTURE / DATA-MODEL CHANGES
════════════════════════════════════════════════════════════

[ ] ★ #8 Length mode overhaul                 → v3.33.0
    Replace single lengthLimit field with explicit mode
    picker on Setup 3:
      • No limit    — silent, no gates
      • Hard cap    — ceiling only, no floor, no
                      undersized modal
      • Target      — strict at N, both floor and ceiling
                      at the target value, symmetric
                      guards armed
      • Range       — between min and max, both fields
                      user-supplied
    Builder prompt envelope must change per mode:
      • Hard cap: "Stay at or below N. Shorter is fine."
      • Target:   "Hit N exactly. Do not deliver less,
                   do not deliver more."
      • Range:    "Stay between min and max. Aim for the
                   middle."
    Migration: pre-v3.32.29 projects default to Hard cap
    mode with their existing lengthLimit copied across.
    Convergence-under and round-undersized intercepts
    only fire in Target or Range modes. Kills the implicit
    LENGTH_FLOOR_RATIO = 0.5 from v3.32.28 — replaced by
    explicit user-opted modes.
    SCOPE: data-model change. Touches Setup 3 UI, prompt
    envelope, all four length-guard intercepts, migration,
    helper-page docs. Probably v3.33.0 since it's a
    model change.

[ ] #1 Fluid scaling architectural pass
    Footer scaling at 1872×938 was the trigger. Symptom
    of a deeper viewport-strategy problem: fixed px vs
    clamp/fluid units, the min-width: 1024px floor, the
    @media (min-width: 1601px) desktop-cascade pattern
    that has produced bug after bug.
    PRECONDITION: dedicated planning chat to produce
    breakpoint inventory and smoke-test matrix BEFORE
    any code changes. Per house rule, CSS deletion sweeps
    require visual smoke testing on every screen before
    main; split into smaller releases each verified
    independently.

[ ] Hive Profiles / saved AI configurations    → 3-phase
    A profile is a saved snapshot of activeAIIds plus
    optionally builder. Dropdown at top of Worker Bee
    screen:
       Profile: [Work Laptop ▾] · Save current as new… · Manage…
    Server-aware: warn instead of silently failing on
    round-start when a profile references AIs from an
    unreachable server (work VPN dropped, home firewall,
    air-gap). Auto-detect via Models endpoint ping.
    Export/import as JSON. API keys redacted on export
    and re-prompted on import.
    Phase 1: trash cans → permanent checkboxes, mode
    toggle removed.
    Phase 2: saved Hive Profiles.
    Phase 3: Sources view formalizing servers as first-
    class entities.

[ ] Sources / Servers view                     ⚠ blocked
    Companion to Hive Profiles (Phase 3). Soft split:
    Worker Bee screen stays the unified "what to use"
    view. New Sources view handles server-level concerns:
    add/edit/remove a server, endpoint pair, API key,
    connectivity test, see which bees came from it.
    Imported Groups becomes "Sources" — earns its keep
    because it's no longer redundant with the bee list,
    it's the parent of bees from that source.
    Export/import maps cleanly: export a Source plus the
    Profiles that reference it.
    BLOCKED ON: Hive Profiles Phase 1+2 shipping first.
