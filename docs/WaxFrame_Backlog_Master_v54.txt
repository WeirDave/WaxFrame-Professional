DO NOT blind-delete — static grep is insufficient. display:contents rules are load-bearing despite no visible effect, and class names built via JS template-literal concatenation evade word-boundary grep (see v3.21.26 regression). Needs a dedicated branch, surgical phased deletion, and visual smoke testing on every screen as a pre-deploy gate.


Observations

- Codex's audit pattern is now repeatable. Pull files into his context, ask for a read-only audit with line numbers and behavior deltas. He delivers a prioritized list with rejection reasoning baked in. Three audits in two sessions, three useful releases out of them. Lean on him whenever the next move is "what did we miss?" rather than "build this feature."

- The docs/ folder now ships in the release zip. This is a meaningful workflow change. Codex was already reading docs/ via GitHub raw URLs, but every other consumer of the zip (offline installs, air-gapped Anduril work laptop) only saw what the zip contained. Now backlog state travels with the release.

- Three-version rollback margin in docs/: keep the last 3 backlog files (v52, v53, v54 as of this release). When the zip ships v54, the local docs/ folder still contains v51 and possibly older — those should be deleted manually after extraction since zip extraction can't remove files. Future automation idea: a docs/ cleanup script that runs post-extract.

- privacy.html and terms.html drifted to a v3.60.8 cache-bust without anyone noticing. Why: the standard release sweep only touches 6 HTML files (index + 5 helpers); legal pages weren't on the list. Decision: privacy.html and terms.html are now part of the standard 8-HTML sweep. Memory rule updated.

- Image references with onerror="this.style.display='none'" silently hide 404s. This is a defense-in-depth that turned into a stealth bug carrier. Three icon files have been broken probably for many versions and no one saw it because the icons just didn't render. Worth a future audit: any onerror hide-on-fail pattern is hiding something.

- The "Open default AI websites" / "Open API Websites" → "Get API keys" rename was a behavior change too, not just a label change. Old behavior: opened multiple browser tabs simultaneously (popup-blocker risk). Current behavior: opens an in-app drawer with clickable provider URLs (no popup-blocker risk). The manual was documenting both the old name AND the old behavior; this release brought both forward.

- Cost ≠ destruction calibration from v3.63.3 reaffirmed — Recommend Models for All carries a suppressKey despite incurring API calls. The rule is about safety brakes on irreversible actions, not about cost. User clicked it intentionally; modal is "did you mean it?" not "you're about to lose data." Locked in.

- Tonight's three-release pattern (docs truth-up → small feature → straggler cleanup) is a good rhythm. Each release was surgical, each had a single coherent theme, and the diffs stayed reviewable. Beats one fat v3.64.0 with all three concerns interleaved.
