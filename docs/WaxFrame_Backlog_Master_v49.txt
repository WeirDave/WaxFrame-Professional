OPEN BUGS / UX
------------------------------------------------------------

- PDF preview after browser refresh shows "Preview isn't available"
  even though the file is still in the project. Because the blob
  URL lives only in memory (window._lastPDFPages / blob in a
  variable), a refresh nukes it. Either:
    (a) reconstruct the blob URL from a stored copy in IDB, OR
    (b) hide the magnifying glass entirely when no blob URL exists
        — don't show a non-functional control. David's instinct:
        why is the magnifying glass even there if clicking it does
        nothing useful?

- Auto-backup misleading in user manual. Either:
    (a) clarify the manual: "WaxFrame auto-saves to this browser.
        Manual backup file required for cross-browser restoration."
    (b) actually implement disk auto-backup (probably overkill —
        browsers can't write files without user click)

- restoreRound only rolls back document state. Does NOT roll back
  console transcript, project timer, token counters, Deep Dive
  capture, or UI accumulator displays. Define what SHOULD roll
  back vs what represents "session history" worth keeping. David
  is surprised when restoring to Round 1 doesn't feel like a
  reset — that perception is the bug, whether or not the behavior
  is technically correct.

TROUBLESHOOTING TOOLKIT — needs definition
------------------------------------------------------------

David asked me to define what each capture mechanism gives us.
Honest current state:

TRANSCRIPT (exportTranscript)
  Has: round-by-round event log, AI responses, USER DECISIONS,
       phase transitions, errors as they happened
  Missing: HTTP-level details, exact prompts sent per round,
           parse diagnostics, token usage per AI per round (unless
           DD is on), the raw model response before parsing

BACKUP (backupSession in storage.js)
  Has: full hive config, project, current docText, history array
       (every round's doc/notes/phase/decisions), resolved
       decisions, settings, IDB session — full pause/resume fidelity
  Missing: any API request/response detail, transcript, debug data

DEEP DIVE
  Has: per-round forensic capture — prompts sent, raw responses,
       parse diagnostics, finish reason, token usage per AI
  Missing: needs verification of what's actually captured today.
  Currently: needs to be confirmed if always-on or toggle-triggered.

The gap between transcript and DD is what David wants to close
with the toggle so troubleshooting doesn't fall in the seam.
