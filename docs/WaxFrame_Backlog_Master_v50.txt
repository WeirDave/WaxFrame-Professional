v3.62.0 — TROUBLESHOOTING TOOLKIT (P0 per David)
  Goal: Someone downloads WaxFrame, hits a problem, can send one
  artifact that gives me everything to diagnose. Without the user
  needing to re-run, screenshot, or guess what we want.

  Pieces:

  1. Standalone troubleshooting page (waxframe_techsupport.html)
     - Reachable even if index.html is broken
     - Reads localStorage/IDB directly with minimal JS (no app.js dep)
     - Hosts: diagnostic bundle button, GitHub Issues link, user
       manual link, FAQ, common error explanations
     - Easy URL to remember and document
     - Bookmark-recommended in user manual

  2. Diagnostic bundle generator
     - One button, three locations: every troubleshooting card,
       Settings menu, troubleshooting page
     - Builds single JSON containing:
       * Session state (scrubbed of secrets + PII)
       * Last N rounds of forensic detail (prompts sent, raw
         responses, parse diagnostics, finish reasons, token usage
         per AI per round)
       * Console log buffer
       * Browser + OS + WaxFrame version + build stamp
       * Specific error context if triggered from an error card
     - Filename: WaxFrame-Diagnostic-{project}-{YYYYMMDD-HHmm}.json
     - Auto-downloads

  3. Always-on forensic ring buffer
     - Currently Deep Dive is intermittent/toggle. For P0 it needs
       to be ALWAYS-ON in a ring buffer (last 5 or 10 rounds default)
     - Memory cost minimal — text data, capped
     - Means a user hitting an error has the forensic data already
       captured, no "go reproduce it with Deep Dive on" round trip

  4. Scrubber function
     - Single utility: takes a session/state object, returns stripped
       version (API keys removed, names/emails optional)
     - Used by:
       * Diagnostic bundle (automatic, always strips)
       * Standalone backup scrubber tool (still on backlog, see
         below — weaker use case now)

  5. GitHub Issues integration (David teaches me this)
     - Replace any "email to support" UX with GitHub Issues
     - Pre-filled new-issue URL pattern:
       https://github.com/WeirDave/WaxFrame-Professional/issues/new?title=...&body=...
     - Body size limit ~8000 chars — diagnostic bundle attached
       manually (download + upload to issue)
     - First-error banner: "Click here to report this — opens a
       GitHub Issue with the diagnostic bundle ready to attach"

  6. UX documentation
     - User manual update explaining the flow
     - Visible banner/hint on first error in any session telling
       new users where to go

OPEN BUGS / UX
------------------------------------------------------------

- PDF preview after browser refresh: magnifying glass opens Verify
  panel but no preview because the blob URL only lives in memory.
  Refresh nukes window._lastPDFPages. Two paths:
    (a) reconstruct blob URL from stored copy in IDB, OR
    (b) hide the magnifying glass when no blob URL exists — don't
        render a non-functional control. David's instinct: don't
        show controls that can't function.

- restoreRound only rolls back document state. Does NOT roll back
  console transcript, project timer, token counters, Deep Dive
  buffer, or UI accumulators. That's why restoring to Round 1
  doesn't feel like a reset. Define what SHOULD roll back vs
  what's legitimate session history.
  This is the reason backups are the actual time machine and
  Round History is just the working tool. Document the distinction.

- Auto-backup misleading in user manual. Need to clarify: WaxFrame
  auto-saves to this browser only (IDB). Cross-browser/cross-machine
  restoration requires a manual backup file. The v3.61.0 checkpoint
  nudge addresses this — but the manual still needs the language fix.

OPEN FEATURES (post-v3.62.0)
------------------------------------------------------------

Custom templates
  - User creates document of a type WaxFrame doesn't ship a template
    for, OR modifies an existing template
  - "Save as my template" button
  - Stored in localStorage or IDB, appears in template picker
    alongside built-ins
  - Reusable across projects

Shareable templates
  - Export a custom template as a portable JSON
  - Import someone else's exported template
  - Different mechanism from backup-sharing — no PII issue, no
    scrubbing needed
  - This is the cleaner answer to "I want to share my workflow
    with a co-worker"

Backup scrubber standalone tool (weakened use case)
  - Originally intended for sharing backups with co-workers
  - Use case is now mostly served by shareable templates
  - Still worth keeping on the list because some backups carry
    actual project state worth sharing (a finished doc the user
    wants a colleague to see the work behind)
  - Scrubbed file CANNOT be imported back into WaxFrame (keys
    missing) — it's a read-only artifact for sharing
  - Naming: Scrubbed-Backup-{original-name}.json
  - Lower priority — only build if real users ask for it

Deep Dive toggle (in Settings)
  - Once v3.62.0 makes Deep Dive always-on as a ring buffer, this
    toggle may become unnecessary. Revisit then.
  - If we still want it, it's the 5th Settings toggle.

Pause/resume at reviewer seam (architectural)
  - Pause AFTER reviewer gather, BEFORE Builder call
  - User fixes broken model via inline picker
  - Hits Resume → re-call just the failed reviewer, merge response,
    proceed to Builder
  - Real round-engine surgery. Spec-first then build.

File System Access API (exploration)
  - Chromium-only (Chrome/Edge) — user grants WaxFrame write access
    to a folder once, then we silently write backup files there
    forever. True Word-style auto-save-to-disk.
  - Doesn't work in Firefox (David's browser of choice) or Safari.
  - Investigate later; not committed. Possibility if WaxFrame ever
    leans into desktop-app feel for Chromium users.

TROUBLESHOOTING TOOLKIT — CAPTURE GAPS (drives v3.62.0)
------------------------------------------------------------

TRANSCRIPT (exportTranscript)
  Has: round-by-round event log, AI responses, USER DECISIONS,
       phase transitions, errors as they happened
  Missing: HTTP-level detail, exact prompts sent, parse diagnostics,
           per-AI per-round token usage, raw model response

BACKUP (backupSession)
  Has: full hive config, project, current docText, history array
       (every round's doc/notes/phase/decisions), resolved decisions,
       settings, IDB session — full pause/resume fidelity
  Missing: any API request/response detail, transcript, debug data

DEEP DIVE
  Has: per-round forensic capture — prompts sent, raw responses,
       parse diagnostics, finish reason, token usage per AI
  Missing: needs verification of what's actually captured today
  v3.62.0 plan: always-on ring buffer (last N rounds) so it's
  ALWAYS available, no "reproduce with DD on" round trip

The diagnostic bundle = scrubbed(BACKUP) + DD-ring-buffer +
console log + browser/OS/version metadata + error context.
Single artifact, single click, ready for a GitHub Issue.
