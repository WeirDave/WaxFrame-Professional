Observations

- v3.63.5 retired the last native browser dialog in the app. The main app went wfConfirm in v3.52.8; the self-contained helper-page license logic was the straggler. Lesson: when migrating a UX pattern, grep for the OLD pattern across ALL files including self-contained helper modules, not just the main app. confirm( and alert( are good grep targets for a periodic sweep.

- The helper-page styled dialog (lhConfirm/lhNotice in license-helper.js) reuses existing .license-modal-* CSS classes rather than adding new ones. This is the right move for self-contained helper modules — lean on CSS that's already loaded on the page rather than shipping new styles. If future helper-page confirms are needed, lhConfirm/lhNotice are now available app-wide on helper pages.

- Comment drift is real and accumulates. Two stale comments this release (app.js toolbar list, manual nav source comment) both described UI states that changed versions ago. Code comments can lie — David has flagged this before. Worth a periodic "do the comments still match" pass, especially around UI that's been renamed.

- Dead-CSS remains the one item that keeps getting deferred, correctly. Every audit flags it; every time the answer is "not as a quick task." The discipline holds: ~155 flagged selectors, some confirmed live, blind delete = regression risk. When it's done it gets its own branch and its own session with visual smoke testing as the gate. Not a bedtime job.

- Four releases in one night, all surgical, all single-theme. The rhythm held: docs truth-up, small feature, straggler cleanup, UX consistency. Each had a reviewable diff and a clean changelog. This is the preferred cadence over fat multi-concern releases.

- docs/ shipping in the zip (since v3.63.4) means backlog state now travels to offline/air-gapped installs. v55 ships with v54 + v53 as the 3-version rollback margin. Remember to delete v52 and older from the local docs/ folder after extraction — zip overlay can't remove files.
