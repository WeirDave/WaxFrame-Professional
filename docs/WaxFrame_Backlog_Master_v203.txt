State: v3.63.134 Pro shipped on main. GitHub Actions release-ceremony validator live + 18 silently-failing CSS token typos fixed (caught by the validator on its first run). New files: .github/workflows/release-check.yml + tools/release-check.mjs (pure Node stdlib, no npm deps — matches WaxFrame's vanilla-no-build-step stance). The validator runs on every push to main and every PR targeting main; same script runs locally via `node tools/release-check.mjs`. Checks: (1) `node --check` on every JS file under js/. (2) Version-stamp consistency: APP_VERSION (from js/version.js) must match package.json `version`, the JSON-LD `softwareVersion` on index.html, every `?v=X.Y.Z` cache-bust ref on every HTML file, every `<meta name="waxframe-build">` stamp, and every `// Build:` comment on every JS file + style.css. (3) CSS token references: for every `var(--TOKEN)` in style.css, TOKEN must be defined either in a :root block or anywhere else as a custom-property declaration (scoped vars, inline overrides set via JS setProperty, animation params in HTML templates). First validator run against main caught 18 silently-failing token refs — all fixed in this same release: --text2 (×4) → --text-dim, --bg2 → --surface2, --bg3 (×2) → --surface3, --text-faint → --text-dim, --amber-dim → --accent-dim, --bg-deep → --bg, --danger → --err, --fs-10 (×4) → --fs-11, --space-7 → --space-6, --space-22 (hp-section margin, separate from the v3.63.131 instance) → --space-24, --space-5 → --space-20, --space-2 → --space-8, --accent-bg → literal rgba, --surface-hover → literal rgba, --text-bright → --text, --surface-elev → literal rgba, --font-body → --font. Some had fallback literals so the rules were silently using the fallback the whole time (defeating the design system); others had no fallback so the property was effectively no-op. Net visual impact: minimal — but the design system is consistent now and future drift fails CI at PR time. SCRIPT DESIGN: regex-based not AST-based (sufficient for WaxFrame's well-formed style.css; avoids npm install dep). Permissive on scoped CSS vars — scans HTML for `style="--foo: …"`, JS for `setProperty('--foo', …)` and template-literal `--foo:` patterns. Catches the typo set without flagging the legitimate ~30 scoped/animation tokens. SCOPE NOT INCLUDED (deferred): dead-link check (every src=/href= to local file exists).
