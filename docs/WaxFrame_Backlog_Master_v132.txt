Open Features

- Builder Failover — primary Builder hits CONTENT_FILTERED / EMPTY_RESPONSE → auto-failover to a configured Secondary Builder for that one round only, then snap back to primary next round. Mirrors how the hive already handles a single reviewer failing. Implementation: "Secondary Builder" pick on the Builder screen, interceptor in `runBuilderOnly()` that swaps to secondary on CONTENT_FILTERED, completes the round, resets. Lets users stay on cheap providers (Gemini Flash, DeepSeek) as primary with a reliable but pricier secondary (Claude/ChatGPT) ONLY firing on blocked rounds. Bigger than the v3.63.66 cluster, well-scoped. Candidate for v3.63.67+.

- Document-playbooks deep-dive entry for Product Review template — queued v3.63.61 alongside template ship. Currently has table-row entry only. Full ~200-line deep-dive section (like the existing entries for Refine / Tighten / Expand) needs writing. Cover: when to use, refMaterial field strategy (the 23-field scaffold), category-specific platform reshapes, headline-directive behavior (v3.63.64), expected convergence rounds (append to playbook-test once measured), and platform-specific length recipes.

  Amazon length recipe (canonical, captured 2026-05-30 from live session):
    Minimum:  500 chars   (under this and the review feels phoned-in)
    Normal:   1,500–3,500 chars   (sweet spot — the majority of useful Amazon reviews)
    Detailed: 4,000–7,500 chars   (long-form, thorough — when you've genuinely used it heavily)
    Hard cap: 20,000 chars   (Amazon's actual ceiling, well past the 4K guidance the field hint cites)
  Note: the template's existing field hint cites Amazon's "Range · 500–4,000" guidance. The 20K hard cap is the actual product limit Amazon will accept; 4K is the recommended ceiling. Document both in the deep-dive so users understand the distinction (recommended vs hard).

  Other platform caps for reference (from existing template hint):
    Best Buy:  5,000 chars (hard)
    Target / Walmart: 4,000 chars (hard)
    Newegg:    5,000 chars (hard)
    Manufacturer site / blog: no limit
  Equivalent recipes for these platforms — TBD; gather from live runs as users actually post to them.
