Observations

- Codex's audit pattern worked clean. He delivered line numbers, before/after behavior deltas, and a vetted candidate list with rejection reasoning. The "I did not edit files" framing kept review scope contained. Worth leaning on him again when an audit is the right starting move — saves a session of grepping.

- Build stamps drift silently between releases. Heading into tonight, index.html + waxframe-user-manual.html + app.js were at -017 while every other file was at -018. The release ceremony sweep catches it, but the gap is invisible until something fires the sweep. Not a bug — just a thing to know.

- Per-template suppress keys (waxframe_suppress_template_confirm__<tpl.id>) is a generalizable pattern. If a future feature needs per-X suppression (per-project, per-AI, per-doc-type), the recipe is: stable slug as suffix, append to a base prefix, the resetSuppressedPrompts prefix sweep catches everything regardless of suffix.

- Recommend Models for All got a suppressKey despite being cost-incurring. The rule "never suppress destructive confirms" is about safety brakes on irreversible actions, not about cost. Cost ≠ destruction. The user clicked the button intentionally; the modal is "did you mean it?" not "you're about to lose data." Calibration locked.

- Total dismiss-able confirm count now 9: Length Guard disable, Length Guard re-arm, Sensitive Backup, Restore Backup, Checkpoint Backup Nudge (per-project), Server/Internet mode switch, Add ref doc mid-session, Recommend Models for All, Go Home with active session, plus per-template educational modals (variable count, one per template with a confirmModal). resetSuppressedPrompts prefix sweep handles them all uniformly.

- The "Modals that do NOT carry the checkbox" list in the manual stayed intact through Release B because none of the 5 new candidates were destructive. The destructive set is stable across this release: Remove API Key, Clear Project, Clear All Reference Material, Remove N Custom AIs, Clear Working Document, "Saved session did not load." If a future audit revisits, those stay un-suppressible.

- v3.63.2 was almost entirely an HTML-copy release. No JS logic changed. Worth remembering that "ship a release" doesn't have to mean code — docs-only releases are legitimate when the docs are wrong about behavior the user relies on.
